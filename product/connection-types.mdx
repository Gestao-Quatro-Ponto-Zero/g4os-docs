## Como pensar nas conexões no G4 OS

No G4 OS, conexões são o que trazem contexto, ferramentas e capacidade de execução para dentro do seu workspace.

Nem toda conexão funciona do mesmo jeito. Algumas são prontas e gerenciadas pelo produto, outras dependem de configuração técnica, e outras já embutem um comportamento de agente.

## Comparação rápida

| Tipo | O que é | Melhor quando | Onde abrir |
| --- | --- | --- | --- |
| Conexões Gerenciadas | Integrações prontas, mantidas pelo G4 OS | Você quer velocidade, autenticação guiada e menos configuração manual | `g4os://sources/g4-managed` |
| MCPs | Servidores que expõem ferramentas via MCP | Você precisa de ferramentas mais flexíveis, locais ou remotas | `g4os://sources/mcp` |
| APIs | Integrações diretas com endpoints e credenciais | Você já tem uma API definida e quer configurar o acesso diretamente | `g4os://sources/api` |
| Agentes G4 | Fontes com comportamento de agente e capacidades específicas | Você quer execução especializada, multimodalidade ou geração | `g4os://sources/g4-agents` |

## Conexões Gerenciadas

Conexões gerenciadas são o caminho mais simples para plugar ferramentas populares no G4 OS.

Elas já vêm com a experiência do produto preparada para autenticação, reconexão, diagnóstico e uso em sessão. Na prática, isso reduz bastante a necessidade de configurar MCP, endpoint, escopo ou runtime manualmente.

### Quando faz mais sentido

- quando você quer conectar apps comuns do dia a dia com o menor atrito possível
- quando o time precisa de uma integração confiável, com fluxo de auth guiado pelo produto
- quando o objetivo é colocar uma operação no ar rápido, sem montar infraestrutura própria

### Onde aparece no app

- `Fontes > Conexões Gerenciadas`
- [Abrir Conexões Gerenciadas](g4os://sources/g4-managed)

## MCPs

MCPs são conexões baseadas em Model Context Protocol. Em vez de um catálogo pronto do G4 OS, você conecta um servidor que expõe ferramentas e capacidades para o produto consumir.

Esse é o caminho certo quando você precisa de algo mais customizado, quando a integração já existe em MCP ou quando faz sentido manter a lógica fora do G4 OS.

### Quando faz mais sentido

- quando sua integração já existe como servidor MCP
- quando você precisa expor ferramentas mais específicas do que uma API pronta
- quando a autenticação, o runtime ou a lógica da integração precisam ficar fora do produto

### Onde aparece no app

- `Fontes > MCPs`
- [Abrir MCPs](g4os://sources/mcp)

## APIs

APIs são conexões diretas com endpoints externos. Aqui o foco é configurar acesso a uma API específica, normalmente com URL, credenciais e parâmetros definidos por você.

É uma abordagem mais técnica e mais flexível do que as conexões gerenciadas, mas com menos abstração do produto.

### Quando faz mais sentido

- quando você já tem uma API interna ou externa bem definida
- quando não existe conector gerenciado para esse sistema
- quando você quer um controle mais direto sobre endpoint, auth e payload

### Onde aparece no app

- `Fontes > APIs`
- [Abrir APIs](g4os://sources/api)

## Agentes G4

Agentes G4 são diferentes de uma conexão comum. Em vez de só conectar um sistema, eles já trazem uma capacidade operacional pronta, como entendimento multimodal, análise de documentos ou geração de imagens.

Eles aparecem como fontes especiais dentro do produto e também têm área própria de catálogo e sincronização em `Configurações > Agentes`.

> **Beta:** agentes gerenciados são uma função beta e podem não estar disponíveis para todos os usuários, contas, workspaces ou builds do G4 OS.

### Quando faz mais sentido

- quando você quer uma capacidade pronta de agente, e não só acesso a dados
- quando o caso de uso pede multimodalidade, análise especializada ou geração
- quando faz sentido usar um catálogo central gerenciado pelo produto

### Onde aparece no app

- `Fontes > Agentes G4`
- `Configurações > Agentes`
- [Abrir Agentes G4](g4os://sources/g4-agents)
- [Abrir Configurações de Agentes](g4os://settings/agents)

## Como escolher sem complicar

- Comece por `Conexões Gerenciadas` quando o sistema já estiver no catálogo do G4 OS.
- Use `MCPs` quando você precisar plugar ferramentas mais customizadas.
- Use `APIs` quando já existir uma integração técnica clara por endpoint.
- Use `Agentes G4` quando o valor principal estiver na capacidade do agente, e não só no acesso à fonte.
