## O que este caso de uso cobre

Este exemplo mostra como o G4 OS pode ser usado para analisar **cohorts de clientes e receita por produto** — cruzando retenção mensal, ticket médio, LTV e churn por cohort de entrada.

O dataset é sintético, calibrado a partir de dados reais de uma empresa de serviços B2B, e foi anonimizado para uso educativo.

### O que tem dentro

| Coluna | Descrição |
|--------|-----------|
| `mes_cohort` | Mês de entrada do cohort |
| `mes_referencia` | Mês da observação |
| `produto` | Produto contratado (Programa Premium, Formação Online, etc.) |
| `clientes_ativos` | Clientes ativos naquele mês |
| `receita_mrr` | Receita recorrente mensal |
| `ticket_medio` | Ticket médio do cohort |
| `retencao_pct` | Taxa de retenção em relação ao mês de entrada |
| `ltv_acumulado` | LTV acumulado até o mês de referência |

### O que o G4 OS vai fazer

Ao abrir o link acima, o G4 OS vai:

1. Fazer o download automático do dataset
2. Calcular retenção por produto e mês de cohort
3. Estimar LTV médio e evolução por cohort
4. Identificar os produtos com melhor e pior retenção
5. Gerar um resumo executivo com os principais insights

## Análises possíveis a partir deste dataset

- **Heatmap de retenção** — quais cohorts retêm mais ao longo do tempo
- **Comparação de LTV por produto** — qual linha de produto gera mais valor no longo prazo
- **Sazonalidade de entrada** — cohorts de determinados meses têm melhor desempenho?
- **Projeção de receita** — extrapolar o MRR esperado com base em taxas históricas de retenção
