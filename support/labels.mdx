## Para que servem as labels

Labels ajudam a organizar sessões com marcadores reutilizáveis e, quando necessário, metadados mais estruturados.

## O que você encontra aqui

- árvore de labels do workspace
- semântica visual, incluindo cores
- regras automáticas de aplicação por padrão de mensagem

## Casos comuns

- criar labels para áreas, projetos e prioridades
- ajustar a hierarquia para facilitar navegação
- usar autoaplicação para reduzir trabalho manual
- combinar labels com a visão de sessões para filtrar contexto

## Caminho no app

- `Configurações > Labels`

## Deep links úteis

- [Abrir Labels](g4os://settings/labels)
- [Abrir Sessões](g4os://allSessions)

## Revisar labels com o assistente

O painel de sessões tem um atalho para abrir uma nova sessão já preparada para revisar a estrutura de labels do workspace.

Ao clicar em **Review labels with assistant** (ícone de edição no cabeçalho do painel de labels), o assistente recebe automaticamente o prompt:

> _Explain the current label structure and auto-apply rules, suggest improvements, and guide me through the update process. Do not edit anything yet; ask for confirmation first._

Você também pode abrir essa sessão diretamente pelo deep link:

- [Revisar labels com o assistente](g4os://action/new-session?input=Explain%20the%20current%20label%20structure%20and%20auto-apply%20rules%2C%20suggest%20improvements%2C%20and%20guide%20me%20through%20the%20update%20process.%20Do%20not%20edit%20anything%20yet%3B%20ask%20for%20confirmation%20first.&send=true&mode=ask&workdir=none)

## Dica prática

Se as labels estão sendo usadas só como lista plana, vale revisar a hierarquia e as regras automáticas. Isso normalmente melhora bastante a navegabilidade.
