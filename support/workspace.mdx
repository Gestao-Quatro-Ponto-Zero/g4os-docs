## O que esta área controla

`Workspace` reúne configurações que pertencem ao contexto ativo, e não ao app inteiro.

## O que você encontra aqui

- nome e identificação do workspace
- modo de permissão padrão
- vínculo com Company Context, quando disponível
- fontes padrão do workspace
- cycling de modos
- recomendações de parceiro
- diretório de trabalho
- pasta raiz de projetos
- configuração de local MCP

## Situações especiais

Quando um workspace está quebrado ou incompleto, esta área também é usada para orientar reparo e restauração do caminho raiz.

## Relação com o onboarding

O onboarding inicial do G4 OS cria o primeiro estado do workspace. É ali que normalmente entram:

- nome do workspace
- idioma e região
- empresa
- perfil de trabalho
- objetivos iniciais
- primeiras conexões

Depois que o workspace já existe, a área de `Workspace` vira o lugar certo para revisar e ajustar as decisões que pertencem a esse contexto específico.

## Casos comuns

- renomear um workspace
- revisar qual diretório de trabalho está ativo
- escolher fontes padrão para novas sessões
- configurar company context para consultas internas
- ajustar a raiz de projetos gerenciados

## Caminho no app

- `Configurações > Workspace`

## Deep links úteis

- [Abrir Workspace](g4os://settings/workspace)
- [Abrir Sessões](g4os://allSessions)

## Dica prática

Se a dúvida for “isso vale para todo o app ou só para este contexto?”, normalmente `Workspace` é a resposta quando o comportamento precisa ficar isolado por projeto, time ou operação.

Veja também [Quickstart](/getting-started/quickstart) para o fluxo inicial de criação do workspace.
