## O que é uma sessão colaborativa

Sessão colaborativa é o fluxo usado quando você quer que outro workspace do G4 OS entre na mesma conversa.

Ela é diferente de um link público. Em vez de só abrir uma página para leitura, o G4 OS cria um convite com duas partes:

- uma `URL de leitura`, que identifica a sessão compartilhada
- um `token de edição`, que autoriza a participação colaborativa

Quem recebe esse convite abre o G4 OS no próprio workspace, escolhe `Importar` e cola o texto completo. A partir daí, a sessão compartilhada passa a existir também naquele ambiente.

## Como funciona na prática

1. Você abre a sessão e entra em `Sessão colaborativa`.
2. O G4 OS gera a `URL de leitura` e o `token de edição`.
3. Você envia o texto completo do convite para a outra pessoa.
4. A outra pessoa abre o G4 OS, escolhe `Importar sessão colaborativa` e cola o convite.
5. O workspace de destino passa a acessar a mesma sessão compartilhada.

## O que cada parte do convite faz

### URL de leitura

A URL de leitura identifica a sessão compartilhada e serve como referência para o conteúdo que será aberto no outro workspace.

### Token de edição

O token de edição é o que libera a participação colaborativa. Sem ele, o convite fica incompleto.

Se o token não estiver mais disponível localmente ou já tiver expirado, o caminho correto é girar o token e gerar um novo convite.

## O que você pode gerenciar depois

Depois que o convite já existe, a própria tela de `Sessão colaborativa` permite controlar o acesso:

- `Copiar convite`: copia novamente o texto completo para convidar outra pessoa
- `Atualizar snapshot`: publica o estado mais recente da sessão compartilhada
- `Girar token`: invalida o token atual e força novos acessos a usarem um convite novo
- `Parar colaboração`: encerra o acesso colaborativo dessa sessão

> Girar o token desconecta quem já estava colaborando e exige reentrada com o novo token.

## Quando usar

- quando outra pessoa precisa continuar a conversa no próprio G4 OS
- quando o contexto precisa circular entre workspaces sem virar só um link de leitura
- quando você quer manter um canal controlado de colaboração sobre a mesma sessão

## Quando não usar

- se a outra pessoa só precisa ler a conversa, prefira [Compartilhamento](/support/sharing)
- se quem vai acessar é você mesmo em outro dispositivo, prefira [Controle remoto](/support/remote-control)

## Requisitos práticos

- a outra ponta precisa estar usando o G4 OS
- o convite deve ser enviado com `URL de leitura` e `token de edição`
- o import acontece dentro de um workspace ativo no app

## Caminho no app

- `Sessões > abrir uma sessão > menu de compartilhamento > Sessão colaborativa`

Sessão colaborativa é o fluxo certo quando duas pontas do G4 OS precisam trabalhar sobre a mesma conversa, com convite controlado, importação dentro do app e revogação explícita quando necessário.
